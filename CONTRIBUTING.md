# Contributing

1. Fork and branch from `main`.
2. Run `./tests/run.sh` before opening a PR.
3. Do not include real IPs, private domains, credentials, personal paths, or customer context.
4. Generic overlay examples are welcome under `examples/overlays/`; personal overlays are not accepted.
5. Shell should be macOS bash 3.2 compatible unless a script explicitly checks for a newer shell.
6. Workflow changes require review evidence in the PR description.

DCO sign-off is required for v0.x. No CLA is required.
