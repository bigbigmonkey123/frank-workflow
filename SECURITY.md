# Security Policy

## Supported Versions

During 0.x, only the latest minor release receives security fixes.

## Reporting a Vulnerability

Use GitHub Security Advisories. Do not open public issues for secrets, authentication bypasses, or supply-chain vulnerabilities. If Security Advisories are unavailable for a fork, maintainers should publish a project-specific private contact before public release.

## Response Targets

- Acknowledge: 7 days
- Initial triage: 14 days
- Fix or mitigation plan: 30 days when feasible

## Scope

In scope: bridge scripts, installer, template rendering, secret scanning, CI rules.
Out of scope: user private overlays, third-party LLM providers, local tmux/CLI clients.
